# 技术设计文档

## 概述

本文档描述主题自动追踪系统的技术实现方案。系统 100% 复用 hotnews 现有数据源和搜索 API，在 viewer.html 中添加主题追踪功能。

## 关键词搜索机制

### 中英文关键词处理

主题的关键词列表可能包含中英文混合，如：`["苹果", "Apple", "iPhone", "Mac"]`

**搜索策略：**
- 每个关键词独立搜索，生成独立的 Keyword_Card
- 搜索时在新闻标题中进行 LIKE 匹配：`title LIKE '%关键词%'`
- 中文关键词匹配中文标题，英文关键词匹配英文标题
- 不做关键词合并或去重（用户可以自行删除不需要的关键词）

**示例：**
```
主题：苹果公司
关键词：["苹果", "Apple", "iPhone"]

生成 3 个卡片：
- 卡片1 "苹果"：匹配 "苹果发布新款MacBook"、"苹果股价上涨"
- 卡片2 "Apple"：匹配 "Apple releases new iPhone"、"Apple Vision Pro"
- 卡片3 "iPhone"：匹配 "iPhone 17 Pro 曝光"、"iPhone sales increase"
```

### 与现有标签系统的区别

| 特性 | 标签系统 (TAG_KEYWORD_MAPPING) | 主题追踪 |
|------|-------------------------------|---------|
| 关键词来源 | 静态配置 + AI 生成 | AI 生成 + 用户编辑 |
| 关键词合并 | 多个关键词合并到一个标签卡片 | 每个关键词独立卡片 |
| 数据关联 | 通过 rss_entry_tags 表关联 | 实时搜索，不存储关联 |
| 用户可编辑 | 否（系统管理） | 是（用户自定义） |

## 数据库设计

### 现有表复用

用户通过主题添加的外部数据源，复用现有的 RSS 管理机制：

| 表名 | 用途 | 主题追踪如何使用 |
|------|------|-----------------|
| `rss_sources` | 系统级 RSS 源 | AI 推荐的 RSS 源添加到此表 |
| `rss_source_requests` | 用户提交的源请求 | 可选：需要审核的源走此流程 |
| `user_rss_subscriptions` | 用户订阅关系 | 用户订阅主题关联的源 |
| `rss_entries` | RSS 内容条目 | 聚合新闻时查询此表 |

### topic_configs 表（新增）

```sql
CREATE TABLE IF NOT EXISTS topic_configs (
    id TEXT PRIMARY KEY,           -- UUID
    user_id TEXT NOT NULL,         -- 用户 ID
    name TEXT NOT NULL,            -- 主题名称，如"苹果公司"
    icon TEXT DEFAULT '🏷️',        -- AI 自动生成的 emoji 图标
    keywords TEXT NOT NULL,        -- JSON 数组，如 ["苹果", "Apple", "iPhone"]
    enabled INTEGER DEFAULT 1,     -- 是否启用
    sort_order INTEGER DEFAULT 0,  -- 排序顺序（复用现有卡片排序机制）
    created_at INTEGER NOT NULL,   -- 创建时间戳
    updated_at INTEGER NOT NULL    -- 更新时间戳
);

CREATE INDEX IF NOT EXISTS idx_topic_configs_user ON topic_configs(user_id);
CREATE INDEX IF NOT EXISTS idx_topic_configs_enabled ON topic_configs(enabled);
```

### topic_rss_sources 表（新增）

关联主题与 RSS 源（复用 rss_sources 表中的源）：

```sql
CREATE TABLE IF NOT EXISTS topic_rss_sources (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    topic_id TEXT NOT NULL,        -- 关联的主题 ID
    rss_source_id TEXT NOT NULL,   -- 关联的 rss_sources.id
    created_at INTEGER NOT NULL,
    FOREIGN KEY (topic_id) REFERENCES topic_configs(id) ON DELETE CASCADE,
    UNIQUE(topic_id, rss_source_id)
);

CREATE INDEX IF NOT EXISTS idx_topic_rss_topic ON topic_rss_sources(topic_id);
```

### 数据源添加流程

```
用户选择 AI 推荐的数据源
        │
        ▼
┌───────────────────────────────────────┐
│ 检查 rss_sources 表是否已存在该 URL   │
└───────────────────────────────────────┘
        │
        ├── 已存在 ──▶ 直接关联到 topic_rss_sources
        │
        └── 不存在 ──▶ 插入 rss_sources 表（source_type='user'）
                              │
                              ▼
                      关联到 topic_rss_sources
                              │
                              ▼
                      同时添加到 user_rss_subscriptions（用户订阅）
```

**source_type 字段说明：**
- `rss` - 系统内置/admin 添加的源
- `user` - 用户通过主题追踪添加的源
- `mp` - 微信公众号源

## API 设计

### 1. 主题配置 CRUD

#### GET /api/topics
获取当前用户的所有主题配置。

**响应：**
```json
{
    "ok": true,
    "topics": [
        {
            "id": "uuid-1",
            "name": "苹果公司",
            "icon": "🍎",
            "keywords": ["苹果", "Apple", "iPhone", "Mac"],
            "enabled": true,
            "rss_sources": ["rss-1", "rss-2"],
            "created_at": 1706745600,
            "updated_at": 1706745600
        }
    ]
}
```

#### POST /api/topics
创建新主题。

**请求：**
```json
{
    "name": "苹果公司",
    "icon": "🍎",
    "keywords": ["苹果", "Apple", "iPhone"],
    "rss_source_urls": ["https://example.com/feed.xml"]
}
```

**响应：**
```json
{
    "ok": true,
    "topic": {
        "id": "uuid-1",
        "name": "苹果公司",
        ...
    }
}
```

#### PUT /api/topics/{id}
更新主题配置。

#### DELETE /api/topics/{id}
删除主题。

### 2. AI 关键词与数据源推荐

#### POST /api/topics/generate-keywords
根据主题名称生成 emoji 图标、关键词，并通过联网搜索推荐真实可用的外部数据源。

**实现机制：**
- 使用 DashScope 的 `enable_search` 参数启用联网搜索
- AI 联网搜索与主题相关的 RSS 源和公众号
- 确保推荐的数据源是真实存在且可访问的

**请求：**
```json
{
    "topic_name": "苹果公司"
}
```

**响应：**

### 3. 主题新闻聚合

#### GET /api/topics/{id}/news
获取主题相关新闻（复用 /api/search）。

**参数：**
- `keyword`: 可选，指定单个关键词
- `limit`: 每个关键词返回的新闻数量，默认 50

**响应：**
```json
{
    "ok": true,
    "keywords_news": {
        "苹果": [
            {"id": "1", "title": "...", "url": "...", "source": "36kr", "published_at": 1706745600}
        ],
        "iPhone": [
            {"id": "2", "title": "...", "url": "...", "source": "IT之家", "published_at": 1706745500}
        ]
    }
}
```

## 正确性属性

### 属性 1：主题配置持久化
- 创建的主题配置必须正确保存到数据库
- 刷新页面后主题配置必须能正确恢复

### 属性 2：关键词搜索一致性
- 每个关键词卡片显示的新闻必须与该关键词相关
- 搜索结果必须与直接调用 /api/search 的结果一致

### 属性 3：用户数据隔离
- 用户只能看到自己创建的主题
- 用户无法访问或修改其他用户的主题配置

## 边界情况与错误处理

### 边界情况
1. **主题名称为空或过长** - 前端验证，限制 1-50 字符
2. **关键词数量** - 最少 1 个，最多 20 个
3. **重复主题名称** - 允许同名主题（通过 UUID 区分）
4. **AI 生成超时** - 设置 30 秒超时，超时后提示用户手动输入
5. **RSS 源验证失败** - 提示用户 URL 无效或无法访问
6. **用户未登录** - 提示登录后才能创建主题

### 错误处理
1. **AI 服务不可用** - 降级为手动输入模式
2. **数据库写入失败** - 回滚事务，提示用户重试
3. **搜索 API 失败** - 显示错误提示，支持重试

## 登录要求

- 创建主题：需要登录
- 查看主题：需要登录（主题是用户私有的）
- 编辑/删除主题：需要登录且只能操作自己的主题
- AI 生成关键词：需要登录（消耗 AI 配额）
