# 2026-02-22 修复记录

## 1. 主题栏目右键删除关键词/数据源不生效

### 问题
在主题卡片标题中右键「删除此关键词」和「删除此数据源」都不生效。

### 原因
两个根因：
1. **缺少 API 端点**：前端 `removeTopicKeyword()` 和 `removeTopicSource()` 调用 `GET /api/topics/{topicId}` 获取当前主题数据，但该端点不存在，返回 404
2. **字段名不匹配**：后端 `get_topic_by_id()` 返回 `rss_sources`（数组），但前端期望 `rss_source_ids`

### 修复
- 在 `topic_api.py` 中新增 `GET /api/topics/{topic_id}` 端点（放在 `/{topic_id}/news` 路由之前）
- 端点返回时做字段归一化：`topic["rss_source_ids"] = topic.get("rss_sources", [])`
- `update_topic` 响应中也加了同样的 `rss_source_ids` 归一化

### 修改文件
- `hotnews/hotnews/web/api/topic_api.py`

## 2. 每日AI早报显示非 AI 内容

### 问题
「每日AI早报」栏目显示了大量非 AI 相关内容（考公、时装周、职业观等），没有按 AI 主题过滤。

### 原因
AI 分类模型（qwen-plus）正常运行，会给每篇文章打 `category`（tech/finance/...）和 `topics`（ai_ml/llm/...）标签。但 `api_rss_brief_timeline` 的 cache miss 查询路径只按 `action=include AND score>=75` 过滤，没有利用 tag/category 做 AI 主题筛选。

`cache_warmup.py` 中有正确的 tag 白名单过滤逻辑（`tag_whitelist: ["ai_ml"]`），但直接查数据库的路径缺失了这个过滤。

### 修复
在 `morning_brief_routes.py` 的 `api_rss_brief_timeline` cache miss 路径中：
- SQL 查询加入 `LEFT JOIN rss_entry_tags` 和 `GROUP_CONCAT(DISTINCT t.tag_id)`
- Python 层面加入与 `cache_warmup` 一致的 tag/category 白名单过滤
- 同时保留 `AI_MODEL`、`DEV_INFRA`、`HARDWARE_PRO` 旧 category 作为兜底

过滤效果：3天内 include 文章从 1603 条降到 653 条（40.7%），内容基本都是 AI/科技相关。

### 修改文件
- `hotnews/hotnews/web/morning_brief_routes.py`

### AI 分类系统现状（供参考）
- 模型：qwen-plus，通过 AIModelManager 自动轮转
- Prompt 版本：`mb_llm_filter_v5_multilabel`
- 已标注：117,257 篇文章，281,442 个标签
- category（大类 10 种）：tech, finance, business, entertainment, sports, health, science, lifestyle, education, other
- topics（预设 20 种）：ai_ml(14122), llm(3726), dev_tools(3125), hardware(4657), robotics(1984), opensource(2093) 等
- 动态发现标签：2599 种，其中 182 个被提升为 active
- attributes（9 种）：breaking, official, deep_dive, opinion, tutorial, free_deal, tool_rec, career, event

## 3. 每经网爬虫未抓取最新数据

### 问题
每经网（custom_rpgccn0）爬虫状态显示 success，但最新文章停在 2月16日，没有新数据入库。

### 原因
爬虫抓取的是金融频道 `finance.nbd.com.cn/columns/119/`，用 `class="f-title"` 正则匹配文章。该频道更新很慢（最新只到 2月14日），而每经网首页 `www.nbd.com.cn` 每天有 30+ 篇新文章，用的是不同的 HTML 结构（`title` 属性而非 `f-title` class）。

### 修复
更新爬虫脚本（需在 Admin 后台手动更新）：
- 新增 `source: "home"` 选项，抓取首页 `www.nbd.com.cn`
- 首页用 `<a href="URL" target="_blank" title="TITLE">` 模式匹配，能抓到 130+ 篇最新文章
- 保留原有频道页解析逻辑（`_parse_column`），通过 `source` 参数切换
- config JSON 改为 `{"source": "home"}`

### 修改文件
- Admin 后台 → 自定义源管理 → 每经网（custom_rpgccn0）的脚本和配置
- `hotnews/.agent/skills/project/create-scraper/examples/nbd_finance.py`（示例代码同步更新）

## 4. 服务器 Git Token 过期

### 问题
部署时服务器 `git fetch` 报 403，原因是 remote URL 中的 GitHub Personal Access Token 过期。

### 修复
重新生成 Fine-grained Token（All repositories, Contents: Read and write），更新服务器 remote URL：
```bash
ssh -p YOUR_SSH_PORT root@YOUR_SERVER_IP "cd ~/hotnews && git remote set-url origin https://<TOKEN>@github.com/sundt/uihash-hotnews.git"
```

### 注意事项
- Fine-grained Token 需要手动添加 Permissions → Contents: Read and write，否则默认无权限
- Token 有效期注意设置足够长，避免频繁过期
