## 1. 主题卡片右键菜单 - 删除功能

### 问题
用户希望在主题卡片上右键可以直接删除关键词或数据源，而不是只能「隐藏」。

### 修复
- **关键词卡片**：右键显示「🗑️ 删除此关键词」，点击后调用 API 从主题中移除该关键词
- **数据源卡片**：右键显示「🗑️ 删除此数据源」，点击后调用 API 从主题中移除该数据源
- 两者都是真正的删除操作，等同于编辑主题去掉勾选保存
- 至少保留一个关键词（否则提示）

### 修改文件
- `hotnews/hotnews/web/static/js/src/platform-reorder.js`
- `hotnews/hotnews/web/static/js/topic-tracker.js`（添加 `data-source-id` 属性）

## 4. 删除主题后 404 错误

### 问题
删除主题后，`localStorage` 中保存的 `tr_active_tab` 还指向已删除的主题，刷新页面后出现 404 错误。

### 修复
- 删除主题时，清除 `localStorage` 中保存的 tab
- 恢复 tab 时，如果主题不存在，清除保存的 tab 并切换到「我的关注」
- 加载主题新闻时返回 404，自动移除 tab 并切换到「我的关注」

### 修改文件
- `hotnews/hotnews/web/static/js/topic-tracker.js`

## 5. 多用户主题泄露问题（安全修复）

### 问题
多人同时在线时，用户 A 可能看到用户 B 的主题。原因是服务端渲染的主题 tab 被 CDN/Nginx 缓存。

### 修复
- **禁用服务端注入主题**：`page_rendering.py` 中的 `_inject_user_topics_as_categories` 函数直接返回，不再注入主题
- **前端完全负责加载**：通过 `/api/topics` API 加载当前用户的主题
- **安全防御**：页面加载时先移除所有服务端渲染的主题 tab，再通过 API 重新加载

### 修改文件
- `hotnews/hotnews/web/page_rendering.py`
- `hotnews/hotnews/web/static/js/topic-tracker.js`

## 9. AI 总结配额用完提示优化

### 问题
用户点击 AI 总结按钮时，如果配额已用完，弹窗会"一闪而过"，没有正确显示错误提示和充值引导。

### 原因
1. 前端错误处理没有针对配额错误做专门处理
2. 后端错误信息不友好："本月使用次数已用完，请等待下月重置"（对付费用户不合适）
3. CSS 样式没有被正确加载

### 修复

**后端错误信息优化** (`hotnews/kernel/user/permission_checker.py`)：
```python
# 修改前
if is_vip:
    return "本月使用次数已用完，请等待下月重置"
else:
    return "本月免费次数已用完，订阅会员可获得更多次数"

# 修改后
if is_vip:
    return "本月额度已用完，充值后可继续使用"
else:
    return "免费额度已用完，充值后可继续使用"
```

**前端配额错误专门处理** (`hotnews/web/static/js/src/summary-modal.js`)：
- 在 catch 块中添加配额错误检测（检测关键词：用完、配额、次数、额度、升级、会员）
- 显示专门的配额用完 UI，包含充值按钮和备选操作（阅读原文、Todo、收藏）

**添加 CSS 样式** (`hotnews/web/static/css/viewer.css`)：
- `.summary-quota-error` 相关样式
- 居中布局、渐变色充值按钮、底部操作按钮、深色模式支持

### 修改文件
- `hotnews/hotnews/kernel/user/permission_checker.py`
- `hotnews/hotnews/web/static/js/src/summary-modal.js`
- `hotnews/hotnews/web/static/css/viewer.css`
- `hotnews/hotnews/web/static/css/features/summary-modal.css`
