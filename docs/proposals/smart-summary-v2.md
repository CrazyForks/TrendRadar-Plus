# 智能总结 V2 方案

## 1. 功能概述

将收藏按钮升级为「一键总结」按钮，点击后弹出总结弹窗，自动识别文章类型并使用对应的深度模板生成结构化总结，同时自动加入收藏。

### 1.1 用户体验

```
首页新闻列表
    ↓ 点击 📝 按钮
弹出总结弹窗（居中大弹窗）
    ├─ 显示加载动画 + "正在分析文章类型..."
    ├─ 自动识别类型（如：技术教程）
    ├─ 显示 "正在生成深度总结..."
    ├─ 展示结构化总结（Markdown 渲染）
    └─ 底部显示：文章类型标签 + 已收藏状态
```

### 1.2 核心价值

- **零配置**：用户无需了解模板，后台自动选择最佳方案
- **深度总结**：不同类型文章使用专业模板，比通用总结更有价值
- **一键操作**：总结 + 收藏一步完成

## 2. 文章类型与模板映射

### 2.1 支持的文章类型

| 类型代码 | 中文名称 | 典型场景 |

| 类型 | 模板特点 | 输出结构 |

| 研究 | 学术严谨，方法论评估 | 核心贡献 → 方法论 → 关键结论 → 批判性思考 |
| 商业 | 战略视角，利益链条 | 信号事件 → 连锁反应 → 数据锚点 → 潜台词解读 |
| 趋势 | 数据支撑，预测依据 | 核心趋势 → 支撑数据 → 驱动因素 → 机会建议 |
| 生活 | 实用可行，具体建议 | 主题 → 核心要点 → 实用建议 → 相关资源 |
| 其他 | 通用知识萃取 | 核心问题 → 关键信息 → 主要结论 → 相关资源 |

## 4. API 设计

### 4.1 生成总结（新 API）

```
POST /api/summary
```

**请求参数**：
```json
{
  "url": "https://example.com/article",
  "title": "文章标题",
  "source_id": "zhihu",
  "source_name": "知乎热榜"
}
```

**响应**：
```json
{
  "ok": true,
  "summary": "## 🎯 核心要点\n...",
  "article_type": "tech-tutorial",
  "article_type_name": "技术教程",
  "favorited": true,
  "news_id": "zhihu-abc123"
}
```

### 4.2 处理流程

1. 验证用户登录 + 速率限制
2. 检查是否已有缓存总结
3. 抓取网页内容（直接 → Jina → ScraperAPI）
4. **AI 分类**：调用 AI 判断文章类型（~100 tokens）
5. **选择模板**：根据类型选择对应的深度提示词
6. **生成总结**：使用专业模板生成结构化总结
7. **自动收藏**：将文章加入收藏列表
8. 缓存总结到数据库

## 8. 成本估算

每次总结调用 AI 两次：
1. **分类**：~100 tokens（输入） + ~10 tokens（输出）
2. **总结**：~2000 tokens（输入） + ~500 tokens（输出）

按 qwen-turbo 价格（约 ¥0.001/1K tokens）：
- 每次总结成本：约 ¥0.003
- 每用户每小时限制 20 次：最多 ¥0.06/用户/小时
