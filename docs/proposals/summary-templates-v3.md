# AI 总结模板优化 V3

## 问题分析

### 当前问题

1. **输出过长**：模板要求太多章节（6-8个），导致总结冗长，用户阅读负担重
2. **表格渲染失败**：观点评论的"逻辑检验"要求表格，但 AI 输出格式不规范
3. **内容截断**：max_tokens=2000 可能不够，长文章总结被截断
4. **尾部不适配**：所有类型都用"学习收获+行动清单"，但新闻类文章硬凑学习收获很尴尬

### 用户期望

- 快速获取核心信息（30秒内读完）
- 重点突出，不要面面俱到
- 根据文章类型给出合适的后续建议
- 简洁但不遗漏关键信息

---

## 优化方案

### 设计原则

1. **精简章节**：每个模板 3-4 个核心章节
2. **控制字数**：总结控制在 300-500 字
3. **去掉表格**：用列表替代表格，避免渲染问题
4. **差异化尾部**：根据文章类型设计不同的结尾
5. **提高 max_tokens**：从 2000 提升到 2500

### 差异化尾部设计

不同类型文章的用户需求不同，尾部应该差异化：

| 文章类型 | 用户核心需求 | 尾部名称 | 尾部内容 |
|----------|--------------|----------|----------|
| 技术教程 | 学习、动手实践 | ✅ 行动清单 | 可以尝试的操作、需要的环境 |
| 产品介绍 | 决策、试用 | ✅ 行动清单 | 如何开始使用、替代方案 |
| 研究论文 | 学习、引用 | ✅ 行动清单 | 值得深入的方向、相关论文 |
| 生活方式 | 尝试、体验 | ✅ 行动清单 | 可以尝试的事项、相关资源 |
| 观点评论 | 思考、形成判断 | 💭 延伸思考 | 值得追问的问题、不同视角 |
| 新闻资讯 | 了解事件、跟进发展 | 📌 后续关注 | 需要跟进的发展、相关背景 |
| 商业财经 | 决策、投资参考 | 📊 关注信号 | 需要盯住的指标、相关标的 |
| 行业趋势 | 把握方向、提前布局 | 🔮 趋势启示 | 可能的机会、需要准备什么 |

## 完整模板设计

### 通用规范（精简版）

```
输出规范：
1. 简洁：总结控制在 300-500 字，突出重点
2. 结构：使用 Markdown 标题和列表，不使用表格
3. 准确：不编造，推测需注明"可能"
4. 链接：产品/工具/公司附带官方链接
5. 术语：生僻术语简短解释
```

### ⚖️ 观点评论（opinion）

**用户需求**：理解作者观点，形成自己的判断

```markdown
## 🎯 核心观点
（一句话：作者认为 XXX，因为 YYY）

## �行 主要论据
1. **论据一**：具体内容 + 支撑数据/案例
2. **论据二**：具体内容 + 支撑数据/案例
3. **论据三**：具体内容 + 支撑数据/案例

## 🔍 观点评估
- **可信度**：高/中/低
  - 理由：数据来源是否可靠、逻辑是否严密
- **潜在偏见**：作者可能的立场或利益相关
- **局限性**：观点可能忽略的方面

## 💭 延伸思考
- 如果这个观点是对的，意味着什么？
- 有哪些反面观点值得了解？
- 这个问题还可以从什么角度思考？
```
