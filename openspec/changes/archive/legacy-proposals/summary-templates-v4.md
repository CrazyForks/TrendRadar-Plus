# AI 总结模板优化 V4

## 设计目标

1. **精准分类**：10 种核心类型，覆盖 95%+ 内容场景
2. **容错设计**：即使分类错误，用户体验影响最小化
3. **可纠正**：用户可手动切换类型重新生成
4. **易维护**：模板结构趋同，差异化控制在尾部

---

## 核心设计原则

### 原则 1：模板结构趋同

所有类型共享相同的核心结构，只有尾部差异化：

```markdown
## 🎯 核心内容（必有，1句话概括）
## 💡 关键要点（必有，3-5个要点）
## 📊 重要信息（可选，数据/时间/人物等结构化信息）
## [差异化尾部]（根据类型变化）
```

**好处**：即使分类错误，用户看到的核心内容结构一致，只是尾部建议不同。

### 原则 2：类型数量精简

从 15 种压缩到 **10 种**，减少 AI 分类出错概率：

| 合并前 | 合并后 | 理由 |
|--------|--------|------|
| 新闻资讯 + 公告声明 | 新闻资讯 | 都是事实陈述，结构相似 |
| 商业财经 + 行业趋势 | 商业分析 | 都关注商业洞察 |
| 技术教程 + 知识科普 | 知识教程 | 都是学习类内容 |
| 书影评论 | 归入观点评论 | 本质都是主观评价 |

### 原则 3：置信度降级

AI 分类时返回置信度，低于阈值使用通用模板：

```python
if confidence < 0.7:
    use_template("general")  # 通用模板兜底
```

### 原则 4：来源预分类提示

根据公众号定位预设类型倾向，提高分类准确率。

## 文章类型体系（10 种）

### 类型总览

| 序号 | 类型名称 | 代码 | 典型场景 | 尾部名称 | 尾部 Emoji |
|------|----------|------|----------|----------|------------|
| 1 | 新闻资讯 | `news` | 突发事件、热点报道、官方公告 | 后续关注 | 📌 |
| 2 | 政策解读 | `policy` | 政府文件、法规分析、监管动态 | 影响与应对 | ⚠️ |
| 3 | 商业分析 | `business` | 财报、投融资、行业趋势、商业案例 | 关注信号 | 📊 |
| 4 | 知识教程 | `tutorial` | How-to、操作指南、概念讲解 | 行动清单 | ✅ |
| 5 | 研究报告 | `research` | 学术论文、研究报告、数据报告 | 延伸阅读 | 📚 |
| 6 | 产品介绍 | `product` | 产品发布、工具推荐、App 评测 | 快速体验 | 🚀 |
| 7 | 观点评论 | `opinion` | 专栏、社论、书评影评、争议话题 | 延伸思考 | 💭 |
| 8 | 人物专访 | `interview` | 对话、访谈、人物故事 | 金句摘录 | 💬 |
| 9 | 资源清单 | `listicle` | Top 10、工具盘点、年度总结 | 资源汇总 | 📑 |
| 10 | 生活方式 | `lifestyle` | 生活技巧、消费指南、健康养生 | 行动清单 | ✅ |
| - | 通用 | `general` | 无法归类或置信度低 | 要点总结 | 📝 |

### 类型判断指南（给 AI 的提示）

```
类型判断优先级：
1. 新闻资讯：有明确时间点的事件报道，强调"发生了什么"
2. 政策解读：涉及政府、法规、监管的内容
3. 商业分析：涉及公司、财务、投资、行业的分析性内容
4. 知识教程：教你"怎么做"的内容，有步骤或方法
5. 研究报告：有研究方法、数据分析、学术引用的内容
6. 产品介绍：介绍具体产品/工具/服务的功能和使用
7. 观点评论：作者表达主观看法，有论点论据
8. 人物专访：以对话形式呈现，聚焦某个人物
9. 资源清单：列举多个资源/工具/推荐的盘点类内容
10. 生活方式：关于生活、消费、健康、娱乐的实用内容

边界模糊时的处理：
- 新闻 + 分析 → 看重点是事件还是分析，事件为主选 news，分析为主选 business
- 教程 + 产品 → 看重点是学技能还是介绍产品，学技能选 tutorial，介绍产品选 product
- 观点 + 人物 → 看形式是对话还是独白，对话选 interview，独白选 opinion
```

## 完整模板设计

### 通用输出规范

```
所有类型必须遵守：
1. 简洁：总结控制在 300-500 字
2. 结构：使用 Markdown 标题和列表，不使用表格
3. 准确：不编造信息，推测需注明"可能"
4. 链接：产品/工具/公司附带官方链接（如有）
5. 术语：生僻术语简短解释
```

## 容错机制设计

### 1. 置信度降级

```python
# article_summary.py

def get_article_type(content: str, source: str = None) -> dict:
    """
    返回格式：{"type": "news", "confidence": 0.85}
    """
    # AI 分类逻辑
    result = classify_article(content, source_hints=SOURCE_TYPE_HINTS.get(source))
    
    # 置信度低于阈值，降级到通用模板
    if result["confidence"] < 0.7:
        return {"type": "general", "confidence": result["confidence"]}
    
    return result
```

