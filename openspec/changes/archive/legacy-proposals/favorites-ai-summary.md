# 收藏文章 AI 总结功能方案

## 1. 功能概述

为收藏的文章提供 AI 智能总结能力，用户点击"总结"按钮后，系统自动抓取文章内容并生成结构化摘要。

### 1.1 用户场景

1. 用户在 HotNews 首页收藏感兴趣的新闻
2. 打开收藏面板，点击某条收藏的"总结"按钮
3. 系统显示加载状态，后台抓取文章并调用 AI
4. 总结完成后，在收藏卡片下方展开显示摘要
5. 摘要会被缓存，下次查看无需重新生成

### 1.2 核心价值

- 快速了解文章核心内容，节省阅读时间
- 结构化输出，便于快速定位关键信息
- 支持多种文章类型的智能适配

## 3. API 设计

### 3.1 生成总结

```
POST /api/user/favorites/{news_id}/summary
```

**请求参数**：无（news_id 在 URL 中）

**响应**：
```json
{
  "ok": true,
  "summary": "## 🎯 核心要点\n...",
  "cached": false,
  "model": "qwen-turbo"
}
```

**错误响应**：
```json
{
  "ok": false,
  "error": "文章内容无法抓取"
}
```

### 3.2 处理流程

1. 验证用户登录状态
2. 查询收藏记录，获取 URL
3. 检查是否已有缓存的总结
4. 若无缓存：
   - 抓取网页 HTML
   - 使用 readability 提取正文
   - 调用 DashScope API 生成总结
   - 存储到数据库
5. 返回总结内容

## 4. AI 总结提示词

参考 hotnews-summarizer 插件的提示词设计，采用结构化输出：

### 4.1 System Prompt

```
你是一位专业的内容分析助手。请对文章进行结构化总结，遵循以下规范：

1. 结构化：使用 Markdown 标题、列表和加粗组织信息
2. 准确性：严禁编造原文不存在的细节
3. 语言：输出语言与原文保持一致，技术术语保留英文
4. 深度：不要只做表面摘要，要挖掘核心价值
5. 链接：涉及的产品/工具尽量附带官方链接
```

### 4.2 User Prompt Template

```
请总结以下文章的核心内容：

## 🎯 核心要点
（用 2-3 句话概括文章主旨）

## 📝 关键信息
1. 
2. 
3. 

## 💡 主要结论
（作者的核心观点或建议）

## 🔗 相关资源
（文中提到的产品、工具或链接）

---
【文章内容】：
{content}
```

## 5. 数据库设计

### 5.1 扩展 favorites 表

```sql
ALTER TABLE favorites ADD COLUMN summary TEXT;
ALTER TABLE favorites ADD COLUMN summary_at INTEGER;
ALTER TABLE favorites ADD COLUMN summary_model TEXT;
```

### 5.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| summary | TEXT | AI 生成的总结内容（Markdown） |

## 7. 限制与保护

### 7.1 速率限制

| 限制项 | 值 | 说明 |
|--------|-----|------|
| 每用户每小时 | 20 次 | 防止滥用 |
| 正文长度 | 8000 字符 | 节省 token |
| 总结缓存 | 永久 | 避免重复调用 |

### 7.2 错误处理

| 场景 | 处理方式 |
|------|----------|
| 网页无法访问 | 返回错误提示 |
| 正文提取失败 | 尝试备用方案或返回错误 |
| AI 调用超时 | 30s 超时，返回错误 |
| AI 返回空内容 | 返回错误提示 |
