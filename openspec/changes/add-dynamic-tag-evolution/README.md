# 动态标签演化系统 - 完整方案

## 📋 方案概览

这是一个完整的标签和关键词系统升级方案，包含三个核心模块：

### 1. 动态标签演化系统
让标签能够自动发现、验证、演化和淘汰，适应内容变化。

### 2. 标签图谱系统
构建标签之间的关系网络，支持多维度分类和智能推荐。

### 3. 用户自定义关键词系统
让用户可以输入自己的关键词，实现个性化内容订阅。

## 🎯 核心问题

### 当前标签系统的局限

1. **标签固定**：62 个预设标签，无法自动适应新话题
2. **粒度不够**：只有 2 层（Category → Topic），无法精准分类
3. **缺乏关系**：标签之间是孤立的，无法表达关联
4. **不够个性化**：用户无法自定义关注的内容

### 用户痛点

- 想关注"DeepSeek"，但没有对应标签
- "AI/机器学习"太宽泛，无法精准筛选
- 想找"免费的开源大模型"，但无法组合查询
- 想追踪"马斯克"的动态，但只能搜索

## 💡 解决方案

### 方案 1：动态标签演化 ⭐⭐⭐

**核心思路**：让 AI 自动发现新标签，通过验证后晋升为正式标签

**关键特性**：
- AI 从新闻中提取新标签
- 候选标签验证（频次、置信度、时间跨度）
- 自动晋升、演化、淘汰
- 完整的生命周期管理

**预期效果**：
- 标签覆盖率：70% → 90%+
- 新话题延迟：7天 → < 1天
- 标签数量：62 → 150+

**文档**：
- `proposal.md` - 详细提案
- `design.md` - 技术设计
- `tasks.md` - 实施计划

### 方案 2：标签图谱系统 ⭐⭐⭐⭐

**核心思路**：构建标签关系网络，支持多维度分类

**关键特性**：
- 标签关系（is_a, related_to, competes_with, enables 等）
- 多层级标签（3+ 层）
- 实体标签（公司、产品、人物）
- 智能推荐（基于 PageRank）

**预期效果**：
- 支持跨维度查询（"免费的开源大模型"）
- 标签推荐准确率提升 30%
- 发现隐藏的内容关联

**文档**：
- `tag-hierarchy-analysis.md` - 深度分析
- `tag-graph-implementation.md` - 实施方案

### 方案 3：用户自定义关键词 ⭐⭐⭐⭐⭐

**核心思路**：让用户输入自己的关键词，实现个性化订阅

**关键特性**：
- 灵活的关键词匹配（精确、模糊、语义）
- 关键词扩展（同义词、相关词）
- 关键词 → 标签转化
- 标签 + 关键词组合查询

**预期效果**：
- 用户满意度提升 50%
- 内容覆盖率接近 100%
- 热门关键词自动成为标签

**文档**：
- `user-custom-keywords.md` - 系统设计
- `keywords-tags-integration.md` - 协同方案

## 🎯 使用场景

### 场景 1：追踪新兴技术

**需求**：关注 DeepSeek 相关新闻

**升级前**：
- 只能搜索"DeepSeek"
- 无法订阅
- 容易遗漏

**升级后**：
```
1. 用户添加关键词："DeepSeek"
2. 系统推荐标签：大语言模型、国产大模型
3. 用户订阅标签 + 关键词
4. 自动匹配相关新闻
5. 关键词热度高 → 自动成为正式标签
```

### 场景 2：精准筛选内容

**需求**：找"免费的开源大模型"

**升级前**：
- 无法实现
- 只能手动筛选

**升级后**：
```
标签：科技 > AI/机器学习 > 大语言模型
属性：免费/优惠, 开源
关键词：免费, 开源, API
    ↓
组合查询，精准匹配
```

### 场景 3：追踪人物动态

**需求**：关注马斯克的所有新闻

**升级前**：
- 只能搜索
- 无法订阅

**升级后**：
```
1. 添加关键词："马斯克", "Elon Musk"
2. 系统推荐标签：商业、科技、汽车
3. 自动扩展同义词
4. 实时推送相关新闻
```
