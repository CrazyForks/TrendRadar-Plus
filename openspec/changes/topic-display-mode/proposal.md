# 主题展示模式优化方案

## 问题描述

当前主题追踪功能存在一个核心问题：**关键词与订阅源内容不匹配**。

### 具体表现

以"读书推荐"主题为例：
- AI 生成的关键词：`豆瓣读书推荐`、`微信读书榜单`、`Kindle年度书单` 等
- AI 推荐的公众号：`湛庐文化`、`新京报书评周刊`、`理想国imaginist` 等

**问题**：这些公众号的文章标题很少包含上述关键词，导致用关键词搜索时几乎匹配不到内容。

---

## 解决方案

### 核心思路

**同时展示关键词搜索结果 + 订阅源最新文章**，利用现有机制自动适配：

- 关键词有匹配 → 显示关键词卡片
- 关键词无匹配 → 不显示（现有机制）
- 订阅源有文章 → 显示订阅源卡片

### 效果示例

| 主题类型 | 关键词匹配 | 订阅源文章 | 最终展示 |
|---------|-----------|-----------|---------|
| 苹果公司 | 大量 | 有 | 关键词卡片 + 订阅源卡片 |
| 读书推荐 | 几乎没有 | 有 | 只显示订阅源卡片 |
| AI 技术 | 较多 | 有 | 关键词卡片 + 订阅源卡片 |

**不需要 AI 判断展示模式，自然就达到了智能效果。**

---

## 数据来源

### 关键词搜索
- 在系统全局新闻（rss_entries）中搜索标题匹配关键词的文章

### 订阅源文章
- 用户创建主题时选择的订阅源，包括：
  1. **AI 推荐的** - 混元模型推荐的公众号/RSS（用户勾选确认的）
  2. **用户手动添加的** - 用户自己搜索添加的公众号/RSS
- 这些保存在主题的 `rss_sources` 字段
- 展示这些订阅源的最新文章

---

## 实现方案

### 后端改动

修改 `/api/topics/{id}/news` 接口，返回数据结构：

```json
{
  "ok": true,
  "keywords_news": {
    "苹果": [...],
    "iPhone": [...]
  },
  "sources_news": {
    "湛庐文化": [...],
    "新京报书评周刊": [...]
  }
}
```

### 前端改动

在主题 tab 中渲染两部分内容：

```
主题卡片区域
├── 🔍 关键词搜索结果（有数据才显示）
│   ├── 关键词1 卡片
│   └── 关键词2 卡片
└── 📱 订阅源最新文章
    ├── 公众号1 卡片
    └── 公众号2 卡片
```

---

## 实现步骤

### Step 1: 后端 API
1. 在 `get_topic_news` 函数中增加查询订阅源最新文章的逻辑
2. 返回 `sources_news` 字段

### Step 2: 前端渲染
1. 修改 `renderTopicNews` 函数
2. 先渲染关键词卡片（现有逻辑）
3. 再渲染订阅源卡片（新增）

### Step 3: 样式优化
1. 订阅源卡片使用不同的图标/颜色区分
2. 可选：添加分隔线或标题区分两部分

## 预期效果

- "苹果公司"主题：显示关键词搜索结果（大量）+ 订阅的科技公众号文章
- "读书推荐"主题：关键词几乎无匹配，只显示订阅的读书公众号最新文章
- 用户无需关心展示模式，系统自动适配
