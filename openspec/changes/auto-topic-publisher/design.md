# 主题自动追踪与发布系统 - 详细设计文档

## 1. 概述

### 1.1 目标
实现特定主题（如苹果公司、特斯拉等）新闻的自动追踪、AI 改写、自动发布到微信公众号草稿，减少人工操作。

### 1.2 核心流程
```
苹果新闻 → 自动追踪 → AI改写 → 草稿发布 → [人工审核] → [手动发布]
   │          │          │          │              │
   │          │          │          │              └── 人工操作
   │          │          │          │
   └──────────┴──────────┴──────────┴── 系统自动化
```

### 1.3 设计原则
- **复用优先**：最大化复用 hotnews 现有能力（搜索、标签、AI、微信凭证）
- **简单可靠**：MVP 阶段保持简单，后续迭代增强
- **人工兜底**：保留人工审核环节，不自动正式发布

### 3.2 现有数据能力

| 数据源类型 | 来源 | 存储位置 | 数据量 |
|-----------|------|---------|--------|
| 热榜数据 | NewsNow API (35+ 平台) | `news_items` 表 | 每日数千条 |
| RSS 订阅 | 用户自定义 RSS | `rss_entries` 表 | 持续增长 |
| AI 标签 | AI 自动提取 | `tags`, `tag_candidates` | 动态发现 |
| 公众号 | 微信公众号 RSS | `rss_entries` | 用户订阅 |

### 3.3 复用搜索 API

主题追踪的数据聚合直接复用现有的 `/api/search` 接口：

```python
# 主题配置本质上就是一个"保存的搜索"
topic_config = {
    "id": "apple",
    "name": "苹果公司",
    "icon": "🍎",
    # 复用搜索参数
    "search_query": "苹果 Apple iPhone Mac iOS",  # 多关键词
    "search_mode": "hybrid",
    # 新增：发布配置
    "auto_publish": True,
    "daily_limit": 2,
    "rewrite_style": "news_brief",
}
```

### 4.3 API 设计

```python
# 主题配置
GET    /api/topic-publisher/topics           # 获取所有主题
POST   /api/topic-publisher/topics           # 创建主题
PUT    /api/topic-publisher/topics/{id}      # 更新主题
DELETE /api/topic-publisher/topics/{id}      # 删除主题

# 新闻聚合（复用搜索）
GET    /api/topic-publisher/topics/{id}/news # 获取主题相关新闻

# AI 改写
POST   /api/topic-publisher/rewrite          # AI 改写新闻
POST   /api/topic-publisher/rewrite/regenerate # 重新生成

# 发布
POST   /api/topic-publisher/publish          # 发布到微信草稿
GET    /api/topic-publisher/history          # 获取发布历史

# 统计
GET    /api/topic-publisher/stats            # 获取统计数据
```

### 4.4 数据库设计

```sql
-- 主题配置表
CREATE TABLE IF NOT EXISTS topic_configs (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    icon TEXT DEFAULT '🏷️',
    search_query TEXT NOT NULL,      -- 搜索关键词
    search_mode TEXT DEFAULT 'hybrid',
    daily_limit INTEGER DEFAULT 2,
    rewrite_style TEXT DEFAULT 'news_brief',
    enabled INTEGER DEFAULT 1,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

-- 发布记录表
CREATE TABLE IF NOT EXISTS topic_publish_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    topic_id TEXT NOT NULL,
    original_title TEXT NOT NULL,
    original_url TEXT NOT NULL,
    rewritten_title TEXT,
    rewritten_digest TEXT,
    rewritten_content TEXT,
    status TEXT DEFAULT 'pending',   -- pending/rewritten/published/skipped
    wechat_media_id TEXT,            -- 微信草稿 media_id
    published_at INTEGER,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (topic_id) REFERENCES topic_configs(id)
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_publish_topic ON topic_publish_records(topic_id);
CREATE INDEX IF NOT EXISTS idx_publish_status ON topic_publish_records(status);
CREATE INDEX IF NOT EXISTS idx_publish_url ON topic_publish_records(original_url);
```

- ❌ 多平台发布（仅微信公众号）
- ❌ 实时推送（定时批量处理）
- ❌ 对话式配置（Phase 1 不做）
