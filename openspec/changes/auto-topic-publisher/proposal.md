# Change: 主题自动追踪与发布系统

## Why
实现特定主题（如苹果公司）新闻的自动追踪、AI 改写、自动发布到微信公众号草稿，减少人工操作。

## 功能拆解

### 1. 主题追踪（信息聚合）
- 定义追踪主题和关键词
- 从多个数据源抓取相关内容
- 按相关性过滤和排序

### 2. AI 内容改写
- 将原始新闻改写为适合发布的格式
- 保留核心信息，优化表达
- 生成标题、摘要、正文

### 3. 微信草稿发布
- 调用微信公众号 API 创建草稿
- 支持图文消息格式
- 保留人工审核环节

### 4. 自动化调度
- 定时检查新内容
- 去重机制
- 发布状态追踪

## 架构选择分析

### 方案 A：在 hotnews 中实现

**现有能力：**
| 能力 | 状态 | 说明 |
|------|------|------|
| RSS 爬虫 | ✅ 完整 | 35+ 平台，定时抓取 |
| 标签追踪 | ✅ 完整 | tag_discovery, tag_promotion |
| AI 总结 | ✅ 完整 | article_summary.py |
| 微信凭证 | ✅ 完整 | wechat_shared_credentials.py |
| 微信公众号 API | ⚠️ 部分 | 有读取 API，无草稿发布 API |
| 定时调度 | ✅ 完整 | rss_smart_scheduler |

**需要新增：**
- 微信草稿发布 API（`/cgi-bin/draft/add`）
- 主题配置管理
- AI 改写模板（不同于总结）

**优点：**
- 复用现有爬虫和数据
- 复用 AI 服务
- 复用微信凭证池
- 统一部署和运维

**缺点：**
- hotnews 已经比较复杂
- 功能耦合度增加

---

### 方案 B：在 topic-explorer 中实现

**现有能力：**
| 能力 | 状态 | 说明 |
|------|------|------|
| 主题分析 | ✅ 完整 | AI 分解主题为子维度 |
| 对话式配置 | ✅ 完整 | 自然语言定义追踪规则 |
| RSS 抓取 | ✅ 基础 | 单独的抓取逻辑 |
| AI 服务 | ✅ 基础 | DashScope 集成 |
| 微信 API | ❌ 无 | 需要全新开发 |
| 定时调度 | ❌ 无 | 需要新增 |

**需要新增：**
- 微信公众号 API 集成
- 微信凭证管理
- 定时调度系统
- AI 改写模板

**优点：**
- 产品定位更匹配（主题追踪）
- 对话式配置体验好
- 独立系统，解耦

**缺点：**
- 需要重复开发微信集成
- 需要重复开发调度系统
- 数据源与 hotnews 重复

---

### 方案 C：混合架构

```
topic-explorer          hotnews
     │                     │
     │  主题配置            │  数据源
     │  对话式交互          │  爬虫系统
     │                     │  AI 服务
     │                     │  微信 API
     │                     │
     └──────── API ────────┘
              │
              ▼
         自动发布服务
```

**思路：**
- topic-explorer 负责主题配置和对话式交互
- hotnews 提供数据源、AI、微信 API
- 通过 API 打通两个系统

**优点：**
- 各司其职，职责清晰
- 复用现有能力
- 灵活扩展

**缺点：**
- 系统间依赖
- 部署复杂度增加

---

## 建议

**推荐方案 A（在 hotnews 中实现）**

理由：
1. **80% 能力已具备** - 只需新增微信草稿 API 和配置管理
2. **数据已在 hotnews** - 不需要跨系统同步
3. **运维简单** - 单一系统部署
4. **后续可扩展** - 如果需要对话式配置，可以后续集成 topic-explorer

**实现路径：**
1. Phase 1：新增微信草稿发布 API
2. Phase 2：新增主题配置管理（简单的 CRUD）
3. Phase 3：新增 AI 改写模板
4. Phase 4：新增自动发布调度
5. Phase 5（可选）：集成 topic-explorer 的对话式配置

---

## Non-goals
- ❌ 自动正式发布（保留人工审核）
- ❌ 多平台发布（仅微信公众号）
- ❌ 实时推送（定时批量处理）
- ❌ 对话式配置（Phase 1 不做）
